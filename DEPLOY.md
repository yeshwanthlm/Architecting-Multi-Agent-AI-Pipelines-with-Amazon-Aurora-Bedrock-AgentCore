# SentinelIQ — Deployment Guide

## Prerequisites

```bash
# Tools required
aws --version          # AWS CLI v2+
terraform --version    # Terraform 1.6+
python --version       # Python 3.12+
```

## Step 1 — Configure AWS credentials

```bash
aws configure
# AWS Access Key ID:     <your key>
# AWS Secret Access Key: <your secret>
# Default region:        us-east-1
# Default output format: json
```

Minimum IAM permissions needed:
- `AmazonRDS*`, `AWSLambda*`, `AmazonVPC*`, `IAMFullAccess`,
  `SecretsManagerReadWrite`, `CloudWatchLogsFullAccess`

---

## Step 2 — Deploy Infrastructure (Terraform)

```bash
cd sentineliq/terraform

# Initialise providers
terraform init

# Preview what will be created
terraform plan

# Deploy everything (~8-12 minutes for Aurora to be ready)
terraform apply -auto-approve
```

What gets created:
| Resource | Details |
|---------|---------|
| VPC | 10.0.0.0/16, 2 public + 2 private subnets |
| Aurora PostgreSQL | Serverless v2 (0.5–8 ACU), engine v15.4 |
| Secrets Manager | DB credentials, auto-generated password |
| Lambda (MCP server) | Python 3.12, VPC-attached, IAM function URL |
| Lambda (bootstrap) | Seeds schema + data, invoked once |
| VPC Endpoints | Bedrock Runtime + Secrets Manager |
| NAT Gateway | Private subnet egress |
| IAM Roles | Least-privilege for Lambda + RDS monitoring |

Terraform will also write `agents/.env` with all connection strings.

## Step 3 — Install Python Dependencies

```bash
# Agents
cd sentineliq/agents
python -m venv .venv
source .venv/bin/activate     # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# UI
cd ../ui
pip install -r requirements.txt
```

## Step 4 — Verify DB was seeded

```bash
# Check bootstrap Lambda result
aws lambda invoke \
  --function-name sentineliq-demo-db-bootstrap \
  --region us-east-1 \
  /tmp/response.json && cat /tmp/response.json
```

Expected: `{"status": "ok"}`

## Step 6 — Launch the UI

```bash
cd sentineliq/ui
source .venv/bin/activate
uvicorn app:app --reload --port 8000
```

Open: http://localhost:8000

### Demo flow in the UI:
1. **Dashboard** — see 3 open cases, 5 flagged transactions, stats
2. **Investigate tab** — click `TXN-20240401-8821` (Romania luxury watch)
3. Watch the 3-agent pipeline animate in real time:
   - Supervisor dispatches Analyst + Risk Scorer in parallel
   - Both agents query Aurora via MCP
   - Supervisor synthesizes → verdict written back to DB
4. See risk score 88/100 CRITICAL, category breakdown, recommended action
5. Try the Account Takeover case (score: 97/100)
6. Check **Audit Log** — every agent action is logged to Aurora

## Troubleshooting

**Aurora not ready yet:**
```bash
# Wait for cluster to be available
aws rds describe-db-clusters \
  --db-cluster-identifier sentineliq-demo-cluster \
  --query 'DBClusters[0].Status'
```

**Bedrock model not enabled:**
```bash
# Enable in the console: AWS Console → Bedrock → Model access
# Enable: anthropic.claude-3-5-sonnet-20241022-v2:0
```

**Lambda cold start timeout:**
The first invocation may take 5-10s. Subsequent calls are faster.
Increase Lambda timeout if needed in `terraform/lambda.tf`.

**VPC connectivity:**
The Lambda is VPC-attached. Ensure the security groups allow
Lambda → Aurora (port 5432) as defined in `networking.tf`.
